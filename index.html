About Me

삼육대학교 메카트로닉스공학과에서 시스템 안전 설계를 공부하며, 예외 상황을 허용하지 않는 인터록(Interlock) 설계의 중요성을 배웠습니다.

이 경험은 자연스럽게 소프트웨어 개발로 이어졌습니다. 하드웨어에서 배운 '안전한 시스템 설계' 철학을 데이터 정합성과 트랜잭션 설계에 적용하고 있습니다.

KH정보교육원에서 풀스택 웹 개발을 수료한 후, 꾸준히 Java, Spring Boot, React 등을 학습하며 실무 역량을 쌓아가고 있습니다.

시스템 설계

"기계에서 오작동을 막는 인터록 원리를,
소프트웨어 아키텍처에 이식합니다."

설계 철학

  • 메카트로닉스 전공에서 배운 인터록(Interlock) 설계 — 특정 조건이 충족되지 않으면 다음 단계로 절대 넘어가지 못하게 하는 물리적 안전 원리를 데이터 처리 로직에 적용합니다.
  • 기능의 완성이 아닌, 예외 상황의 완벽한 통제가 개발의 본질이라 생각합니다.
  • 기계의 유기적인 메커니즘을 조망하던 습관을 소프트웨어에 이식하여, 전체 시스템이 유기적으로 맞물려 돌아가게 설계합니다.

실전 적용

  • 캠핑장 예약 플랫폼에서 운영 DB와 격리된 유령 테이블 구조를 독자적으로 설계
  • 모든 데이터가 완벽히 수집된 후에만 트랜잭션을 실행하는 원자적 이관 로직 구축
  • 수십 차례 강제 종료 시뮬레이션을 거쳐 단 한 건의 오차도 없는 시스템 완성
🔒

데이터 정합성

"유령 데이터 발생률 0%.
단 한 건의 오차도 허용하지 않습니다."

핵심 철학

  • 예약 시스템은 금전 거래와 직결 — 단 하나의 데이터 오차도 고객 신뢰를 잃고 비즈니스 손실로 이어지는 치명적 결함입니다.
  • 사용자가 예약 도중 브라우저를 닫거나 네트워크 이탈 시 발생하는 데이터 누락을 원천 차단하는 설계를 추구합니다.
  • AI 도구가 제안한 표준 로직의 한계를 직접 발견하고, 로직의 밑바닥부터 직접 설계하여 해결합니다.

실전 성과

  • 실제 운영 DB와 완전히 격리된 임시 저장 공간에서 검증 후 이관하는 2단계 트랜잭션 구축
  • 삭제 시 데이터 유실 방지를 위한 논리적 삭제(Soft Delete) 적용
  • FK/PK 관계에서 발생하는 고아 데이터 문제를 사전에 차단하는 방어적 DB 설계

논리적 사고

"코드 한 줄에 항상 분명한 이유가 담겨 있습니다.
이유가 분명할 때, 무섭게 몰입하는 사람입니다."

논리 근육 만들기

  • 2년간의 공백기 동안 매일 고밀도 학습 — Java 메모리 관리, 객체 지향 원리를 스스로 납득할 때까지 파고들었습니다.
  • 프로그래머스 코딩 테스트 주 5회 이상 풀이, 비전공자임에도 Lv.4 문제까지 독력으로 해결합니다.
  • 의도적으로 AI 코딩 어시스턴트를 배제하고, 모든 로직을 직접 구현하며 코드가 실행되는 매 순간의 이유를 찾는 데 집중했습니다.

주변의 평가

  • "논리적인 사람" — 행동과 코드에 항상 분명한 근거가 있다는 평가
  • "편협되지 않은 생각" — 서로의 이야기를 들어보고 중심을 잡는 소통 스타일
  • 문제를 구조화하고, 비즈니스의 복잡한 예외를 명쾌한 로직으로 풀어내는 해결사
